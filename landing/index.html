The content distribution problem

You write a great post. Then you copy-paste it to Twitter, LinkedIn, DEV.to, Mastodon... reformatting each time. Every. Single. Time.

Before
  • ✗ Write post in your editor
  • ✗ Manually reformat for Twitter's 280-char limit
  • ✗ Rewrite again for LinkedIn's tone
  • ✗ Convert Markdown for DEV.to front matter
  • ✗ Open 6 browser tabs, paste everywhere
  • ✗ Forget Mastodon. Again.
~45 minutes per post · Error-prone · Soul-crushing
After
  • ✓ Write once in Markdown
  • ✓ Auto-adapt to every platform's format
  • ✓ Publish to all 6 platforms in one command
  • ✓ Schedule for optimal posting times
  • ✓ Track status across platforms
  • ✓ All from your terminal
~30 seconds · Consistent · Automated

Pipeline automation

Automated content pipelines with approval flows and cron scheduling.

→
↓

AI or human
→
↓

Gate check
→
↓

All platforms
contentforge.toml
# Pipeline with cron scheduling
[pipeline.weekly-roundup]
trigger = "cron: 0 9 * * MON"
steps = ["adapt", "review", "approve", "publish"]
targets = ["twitter", "linkedin", "devto"]
approval = "manual" # or "auto"
# Encrypted credentials
[credentials]
store = "encrypted" # AES-256-GCM
